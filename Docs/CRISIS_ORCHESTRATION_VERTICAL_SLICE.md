# 팀 기반 위기대응 훈련 버티컬 슬라이스

## 목적

이 모듈은 교사를 향한 욕설과 물건 던지기가 발생했을 때, 교사가 학생과 일대일로 잘 말하는 능력만이 아니라 **교사 자신·초점학생·주변 학생·지원인력·사후 회복을 함께 조율하는 능력**을 연습하도록 만든 첫 번째 확장 시나리오다.

교사가 순간적으로 느끼는 분노, 당황, 무력감에는 점수를 부여하지 않는다. 평가는 감정을 숨겼는지가 아니라 그 상태에서 선택한 관찰 가능한 행동에 근거한다.

## 학습 흐름

1. **교사 자기상태 확인:** 신체 각성과 감정을 알아차리고, 거리·출구·손·주변 학생을 확인할 판단 여유를 만든다.
2. **학급 안전 확보:** 수업 지속보다 주변 학생과 통로 안전을 우선하고 구경 자극을 줄인다.
3. **구체적 지원 요청:** 위치, 현재 위험, 필요한 역할을 포함해 동료 또는 관리자에게 요청한다.
4. **도착 확인과 인계:** 요청 전송과 실제 도착을 구분하며, 관찰 사실·위험·이미 한 조치·다음 역할을 짧게 인계한다.
5. **저자극 재접촉:** 팀이 확보된 뒤 짧은 정서 인정과 안전한 선택권을 제공한다.
6. **객관적 기록:** 즉각적 위험이 낮아진 후 시간, 직접 관찰, 학생 발화, 조치와 인계를 진단·비난 없이 기록한다.
7. **교사 회복:** 동료 디브리핑, 안정 시간, 상담 또는 업무 조정을 조직적 후속조치로 계획한다.

시나리오의 안전 경계는 명시적이다. 이 모듈은 학생을 신체적으로 제압하거나 교사가 혼자 강제 분리하는 연습이 아니며, 실제 학교의 승인된 안전 절차가 시뮬레이션 규칙보다 우선한다.

## 코드 없이 수정 가능한 콘텐츠

`Assets/Resources/Training/Orchestration` 아래의 세 ScriptableObject가 연구자용 저작 단위다.

- `TeacherDirectedAggression.asset`: 상황, 관찰 신호, 교사 내적 프롬프트, 선택 행동, 실제 발화 예시, 근거, 디브리핑 질문
- `CrisisOrchestrationPolicy.asset`: 높은 각성·위험, 낮은 물리적 안전, 기록 가능 위험의 초기 임계값
- `CrisisOrchestrationAssessmentModel.asset`: 8개 역량과 긍정·주의 근거의 연결 및 점수 규칙

에셋을 다시 생성해야 할 때는 Unity 메뉴 `Teacher Training > Content > Generate Crisis Orchestration Scenario`를 사용한다. 임계값과 점수는 현재 설계 초기값이며, 교사·학교 안전 전문가의 인지면담과 플레이테스트를 거쳐 고정해야 한다.

## ECD 역량

| 역량 | 관찰 근거 예시 |
|---|---|
| 교사 자기조절 | 직접 개입 전 판단 여유 확보 |
| 위험 판단 | 교사 상태와 주변 학생 위험 확인 |
| 지원 요청 | 위험 수준에 맞춘 적시·구체적 요청 |
| 학급 안전 조율 | 주변 학생 이동과 통로 확보 |
| 팀 조정과 인계 | 도착 확인, 사실·위험·조치·다음 역할 인계 |
| 보호자 협력 소통 | 후속 슬라이스에서 구현; 현재는 근거를 추정하거나 점수화하지 않음 |
| 객관적 기록 | 안전 확보 후 시간순 관찰 기록 |
| 사건 이후 회복 | 동료 디브리핑과 심리·업무 지원 계획 |

미관찰 역량은 0점으로 남으며, LLM이나 시스템이 행동 근거를 만들어 내지 않는다. 긍정 근거와 주의 근거는 모두 디브리핑에 보존된다.

## 실행 화면

일반 훈련 HUD의 왼쪽 `안전 훈련` 버튼에서 모달을 연다. 각 비트는 상황·관찰 신호·교사의 내적 질문·실행 가능한 발화가 한 화면에 나타난다. 행동 직후 안전 효과를 설명하고, 완료 시 강점·다시 볼 지점·조직에 요청할 지원을 분리해 보여준다. 같은 HUD 캔버스를 사용하므로 데스크톱 포인터와 Quest XR 레이 입력이 동일한 콘텐츠를 조작한다.

## 연구 로그

표준 `TrainingTelemetryEvent` 스키마 7의 `orchestration` 필드에 다음이 저장된다.

- 시나리오·비트·행동 ID와 수용 여부
- 전후 위기대응 단계
- 전후 교사 각성, 초점학생 위험, 위험구역의 주변 학생 수
- 동료·관리자 응답 상태
- 해당 선택으로 발생한 관찰 근거 ID

교사의 감정 서술 원문이나 학생·교사 대화 원문은 이 모듈이 추가로 저장하지 않는다.

## 검증

EditMode 자동검사는 도메인 전이, 방어적 상태 복사, 너무 이른 개입·기록·인계, 전체 7단계 완료, 8개 역량, 감정 비채점, 시나리오 콘텐츠 범위, 비트 외 행동 거부, 비난 없는 디브리핑을 확인한다. 2026-07-22 선택 회귀 묶음은 `30/30`을 통과했다.

## 다음 콘텐츠 슬라이스

1. 동료가 응답할 수 없을 때 관리자·상담 인력으로 재요청하는 분기와 실제 대기시간
2. 보호자 연락 전 준비, 사실·해석 구분, 비난 없는 협력 대화와 기록
3. 수업 재구성, 다른 학생 설명, 루머·낙인 방지, 다음날 학급 복귀
4. 반복 노출 교사의 누적 피로와 관리자 지원 가용성을 포함한 장기 세션 상태
5. 현직 교사 인지면담을 통한 임계값·발화·루브릭 내용타당도 검토
