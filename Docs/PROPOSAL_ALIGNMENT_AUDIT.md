# 제안서 목표 부합성 점검

점검 대상: 「정서행동위기 학생 대응 역량 강화를 위한 생성형 AI 통합 지능형 시뮬레이션 개발 V3」와 현재 Unity 프로토타입

## 결론

현재 시스템은 **1차년도 기술 프로토타입으로는 방향이 잘 맞지만, 연구용 시스템으로는 부분 부합**한다. 실제 교실, 생성형 AI 학생, 직접 대화, 정서 상태 변화, 대응 선택과 피드백은 제안서의 핵심 경험을 구현했다. 반면 제안서가 요구하는 초등학교 맥락, ECD 기반 역량·근거 모델, STT/TTS, 다중 페르소나, 멀티모달 로그, 개인 대시보드, 웹 기반 접근, 한·영 버전, 심리적 안전장치는 아직 부족하다.

- 1차년도 초기 기능 프로토타입 부합도: 약 78%
- 3개년 전체 연구목표 대비 현재 완성도: 약 48~55%
- 판정: **1차년도 연구 프로토타입으로는 대체로 적합. 이후 스프린트는 실제 교사 검토, 음성 상호작용, WebGL/XR 배포, 타당화 연구 설계로 확장 필요**

## 연구 일정 관점의 해석

현재 시점은 제안서 일정상 1차년도 초반이며, 2026년 6~9월의 주요 과업은 문헌·교육과정 분석, 설계 메커니즘 도출, ECD 프레임워크 개발이다. 실제 시나리오 면담과 스토리보드는 2026년 10~11월, 본 개발은 2026년 12월 이후로 계획되어 있다. 따라서 현재 프로토타입은 기술 일정상 앞서 있지만, 전문가·교사 면담과 ECD 결과가 나중에 나오면 구조를 다시 바꿔야 할 위험이 있다.

## 다음 개발 우선순위

1. **연구 정렬:** 초등학교 대상 3~5개 위기 유형을 확정하고, 각 시나리오를 ECD의 역량·근거·과제·관찰변수와 연결한다.
2. **데이터 스키마:** 교사 발화, 선택, 반응시간, 카메라 시선, 학생 정서 벡터, AU, 제스처, 학급 상태를 타임스탬프 이벤트로 기록한다.
3. **음성 상호작용:** Whisper 계열 STT와 한국어·영어 TTS를 추가하고, 교사의 음량·속도·휴지 같은 비언어 특징을 기록한다.
4. **다중 페르소나:** 연령, 위기 유형, 촉발 요인, 관계 이력, 보호 요인을 포함한 검증 가능한 학생 프로필을 ScriptableObject로 분리한다.
5. **심리적 안전:** 항상 보이는 일시정지·중단, 콘텐츠 고지, 디브리핑, 위기 escalation 규칙을 추가한다.
6. **학습 대시보드:** 정서 변화 그래프, 개입 타임라인, 전략 유형, 놓친 신호, 재시도 권고를 시각화한다.
7. **배포·국제화:** WebGL 빌드와 한·영 리소스 분리 후 한국·미국 문화 맥락별 시나리오를 검토한다.

## 유지해야 할 현재 강점

- Action Unit 기반 미세 표정 제어와 정서가의 연속 전환
- 문제행동 제스처와 학생 반응의 연결
- 학생과 마주 보는 대화 카메라, 아이컨택 상태, 반투명 말풍선
- LLM 장애 시 훈련을 중단하지 않는 로컬 폴백
- 선택 결과 저장, 자동 PlayMode 검증, 재현 가능한 씬 빌더
