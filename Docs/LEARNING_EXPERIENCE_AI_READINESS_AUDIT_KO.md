# 학습경험·AI 러닝서포트·기록 인출 준비도 감사

기준 문서: `정서행동위기 학생 대응 역량 강화를 위한 생성형 AI 통합 지능형 시뮬레이션 개발V3.pdf`
감사 범위: 2026-07-22 현재 Unity 클라이언트, 시나리오·ECD 데이터, OpenRouter/Cloudflare 경계, 연구 로그, 디브리핑 UI

## 결론

현재 시스템은 **1차년도 연구용 프로토타입으로서는 목표에 대체로 부합**한다. 실제적인 한국 학교 공간, 음성·텍스트 자유대화, 학생의 정서·행동 변화, 동적 시나리오 전환, ECD 근거 수집, 성찰과 디브리핑까지 학습 루프가 연결되어 있다. 특히 단순 선택형 시뮬레이션을 넘어 자유 발화가 학생의 정서 벡터·제스처·표정·다음 단계에 영향을 주도록 만든 점은 제안서의 “실시간 적응형 지능형 시뮬레이션” 방향과 잘 맞는다.

다만 **교육적 효과가 입증된 상태는 아니다**. 현재 강점은 구현 준비도이며, 전문가 타당화·사용성·인지부하·전이 효과 데이터는 아직 필요하다. 또한 현재의 문서 문맥 주입과 최근 대화 상태는 RAG의 일부 토대일 뿐, 검증된 지식베이스를 의미 기반으로 검색하거나 이전 세션을 장기 인출하는 체계는 아니다.

내부 감사 기준으로 보면 1차년도 기능 정합성은 약 70–75% 수준이지만, 효과성 근거 성숙도는 약 25–35% 수준이다. 전자는 “기능이 연결되어 있는가”, 후자는 “교사의 실제 역량 향상을 입증했는가”를 뜻하며 서로 다른 수치다.

## 현재 학습경험의 효과 가능성

학습 루프는 `prebrief → 관찰 → 교사 개입 → 학생의 자연적 반응 → 정서/행동 변화 → 성찰 → 근거 기반 디브리핑 → 재시도`로 구성된다. 이 구조는 설명을 읽고 정답을 고르는 연수보다 실제적 판단과 반복 연습에 유리하다. 특히 학생 반응이 교사 발화의 질에 따라 달라지고, 실패해도 다시 시도할 수 있다는 점은 심리적으로 안전한 deliberate practice에 적합하다.

가장 큰 위험은 세 가지다.

1. **HUD 과잉과 주의 분산**: 교사에게 학생 얼굴·몸짓·주변 학생을 보게 해야 하는데, 관찰 텍스트·선택지·말풍선·상태 정보가 동시에 주의를 끌 수 있다. Quest에서는 읽기 각도와 시야 점유가 인지부하를 더 키울 수 있다.
2. **정답 코칭 효과**: 대응 원칙이나 추천 문구가 행동 전에 과도하게 노출되면 실제 판단이 아니라 화면 단서 추종을 학습할 수 있다.
3. **평가의 순환 논리**: LLM이 학생 반응을 만들고 같은 계열의 LLM이 교사 발화를 채점하면, 실제 현장 타당성보다 모델 선호에 맞춘 점수가 강화될 수 있다.

따라서 수행 중 지원은 항상 보이는 큰 패널보다 다음의 점감형 구조가 적합하다.

- 0단계: 지원 없음. 학생·학급 단서만 관찰한다.
- 1단계: 사용자가 요청하거나 장시간 멈추면 “무엇을 볼지”만 알려준다.
- 2단계: 두 번 연속 핵심 신호를 놓치면 관련 대응 원칙을 한 문장으로 제시한다.
- 3단계: 행동 후에만 대안 발화 두 개를 비교하고 근거를 보여준다.
- 재시도: 같은 힌트는 자동으로 줄이고, 새로운 상황에서 독립 수행을 확인한다.

## RAG는 어디에 쓰면 효과적인가

RAG는 학생 NPC가 더 장황하게 말하게 만드는 용도보다 **교사 코칭의 근거성과 추적 가능성**을 높이는 데 쓰는 것이 좋다.

### 권장 지식 소스

- 연구진이 승인한 위기 대응 원칙과 학교 절차
- ECD 역량·관찰행동·근거 정의
- 시나리오별 촉발 요인, 학생 강점, 지원 요구, 금기 대응
- 전문가가 작성한 성공·위험 사례와 해설
- 전자칠판의 현재 수업 자료 문맥

각 문서 조각은 `source_id`, 버전, 승인자, 적용 연령, 위기 단계, 관련 역량, 금기 조건을 가져야 한다. 생성된 코칭에는 사용한 `source_id`를 남겨야 하며, 근거가 없으면 일반 조언을 확정적으로 제시하지 않아야 한다.

### 실시간 경로 분리

- **학생 반응 경로**: 현재 시나리오, 페르소나, 최근 대화, 정서 상태만 사용한다. 지식 검색으로 학생 발화가 교과서처럼 변하는 것을 막는다.
- **행동 후 코치 경로**: 현재 교사 행동과 ECD 관찰 결과를 질의로 사용하여 승인된 근거 2–4개를 검색한다.
- **디브리핑 경로**: 세션 전체의 패턴과 이전 유사 시도의 요약을 검색해 변화와 반복 오류를 보여준다.

현재 활성 PDF 페이지 2,400자와 최근 8턴을 붙이는 방식은 “bounded context injection”이다. 의미 검색, 문서 조각 랭킹, 출처 ID, 검색 로그가 추가되어야 연구에서 RAG라고 부르는 편이 정확하다.

## 장기 기록 인출 설계

원시 대화나 원시 시선을 LLM에 보내지 않고도 개인화는 가능하다. D1에 다음과 같은 비식별 에피소드 요약을 저장한다.

```text
scenario_id, beat_id, crisis_stage, competency_scores,
observable_evidence_ids, missed_signal_ids, affect_delta,
response_latency, gaze_summary, hint_level, retry_number,
prompt_version, model_id, rubric_version
```

다음 세션에서는 “같은 학생”을 기억시키기보다 `유사 위기 단계 + 같은 취약 역량 + 비슷한 놓친 신호`의 과거 2–3개 시도를 인출해 자기 비교를 제공하는 것이 안전하다. 원시 시선 NDJSON은 승인된 오프라인 연구 분석용으로 유지하고, 실시간 OpenRouter 프롬프트에는 보내지 않는다.

## 다음 구현 우선순위

1. **적응형 러닝서포트 상태기계**: 요청·지연·놓친 신호·재시도 횟수로 힌트 단계를 결정하고 사용 이력을 로그화한다.
2. **검증 지식 코퍼스와 검색 출처 계약**: 문서 조각, 메타데이터, 버전, 승인 상태, 인용 ID를 정의한다.
3. **세션 요약·장기 학습자 모델**: D1에 비식별 에피소드 요약을 추가하고, 유사 과거 시도 검색 API를 만든다.
4. **오케스트레이션 과제 강화**: 초점 학생뿐 아니라 또래 안정도, 소음, 수업 참여, 안전 우선순위를 독립 상태로 모델링한다.
5. **평가 타당화**: LLM·규칙 점수와 전문가 블라인드 평정의 일치도, 누락·과대평가 사례를 분석한다.
6. **Quest 수용성 검증**: 실제 헤드셋에서 각도별 텍스트 가독성, 마이크 권한, 20–30분 성능·열, 멀미, 중단 버튼 접근성을 시험한다.
7. **플랫폼 결정 기록**: 제안서의 웹 기반 VR을 유지할지, Quest 네이티브를 주 연구 경로로 변경할지 연구진이 명시한다.

## 효과 검증 설계

단순 전후 점수보다 **보지 못한 새 시나리오로의 전이**를 1차 결과로 두는 것이 좋다.

- 조건 A: 현재 시뮬레이션
- 조건 B: 현재 시뮬레이션 + 적응형 점감 스캐폴딩
- 조건 C: 조건 B + 근거 출처가 표시되는 RAG 코칭

주요 결과는 전문가가 블라인드 평정한 위기 대응 수행, 안전상 핵심 누락, 감정 인정·선택권·수업 복귀 연결, 새 시나리오 전이, 2–4주 지연 유지다. 보조 결과는 반응 시간, 학생 정서 변화, 시선 분배, 힌트 의존도, SUS, IPQ, NASA-TLX, 교사 효능감이다. LLM 점수는 단독 종속변인으로 쓰지 않고 인간 평정과의 일치도·오류 분석을 함께 보고해야 한다.

가장 가치 있는 다음 실험은 **“항상 보이는 지원 패널” 대 “요청·오류 기반 점감 지원”** 비교다. RAG 자체의 효과보다, 언제 어떤 근거를 얼마나 보여주는지가 학습 전이와 인지부하에 더 큰 영향을 줄 가능성이 높다.
