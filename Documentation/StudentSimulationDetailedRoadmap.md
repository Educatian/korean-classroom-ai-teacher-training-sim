# 학생 NPC 시뮬레이션 세부 개발과제

## 1. 한국 초등학생 외형 체계

- 얼굴 비율 프리셋을 머리 폭·높이·깊이, 턱 폭, 볼 볼륨, 눈높이로 분리한다.
- 프리셋 조합이 표정 Action Unit과 충돌하지 않는 허용 범위를 정의한다.
- 자연 흑발·짙은 갈색만 사용하는 헤어 팔레트와 단발, 보울컷, 사이드파트, 포니테일, 트윈타이 프리셋을 확장한다.
- 티셔츠, 맨투맨, 조끼, 카디건, 체육복 색상군을 실제 교실 사진 비율에 맞춰 배정한다.
- 같은 좌석 주변에 같은 얼굴형·옷색·헤어가 연속되지 않도록 배치 제약을 둔다.

## 2. 시선과 주의 전환

- 교사 이동, 교사 발화, 전자칠판 변화, 다른 학생의 문제행동을 시선 자극으로 구분한다.
- 학생별 주의 지속시간, 반응 지연, 회피 확률, 재주시 간격을 성향값으로 관리한다.
- 머리 회전과 눈동자 회전을 분리하고 최대 회전각과 전환 속도를 제한한다.
- 교사가 시야 밖에 있거나 다른 학생에게 가려졌을 때 추적 강도를 낮춘다.
- 여러 학생이 동시에 교사를 보는 비율을 상황별 목표 범위로 검증한다.

## 3. 제스처 레이어

- 기본 자세, 손 제스처, 상체 긴장, 시선, 얼굴 AU를 독립 레이어로 분리한다.
- 중립, 경청, 필기, 몸 뒤척임, 손 만지작거림, 책상 두드리기, 회피, 방어, 항의, 회복 제스처군을 확장한다.
- 같은 클립을 사용하더라도 시작 프레임, 속도, 강도, 좌우 손, 다음 전환시간을 학생별로 다르게 한다.
- 문제행동 강도가 높아질 때 반복 빈도와 움직임 범위가 연속적으로 증가하도록 연결한다.
- 발화 중 제스처와 침묵 중 제스처가 충돌하지 않도록 우선순위를 정의한다.

## 4. 교사 대응 훈련 세부과제

- 감정 인정, 선택권 제공, 요구 수준 조절, 원인 탐색, 안전 확보를 독립 평가 항목으로 기록한다.
- 같은 교사 문장도 학생의 신뢰도·각성도·지배감에 따라 다른 반응을 생성한다.
- 즉시 효과와 누적 효과를 분리해 한 번의 좋은 대응만으로 위기가 완전히 해소되지 않게 한다.
- 부적절한 대응 뒤 회복적 재시도를 허용하고 변화 경로를 디브리핑에 표시한다.
- 관찰 모드에서 시선, 자세, AU, 발화 지연, 문제행동 빈도의 근거를 시간순으로 제시한다.

## 5. 소리와 공간감

- 버튼 클릭, 실내화 좌우 발걸음, 책상 접촉, 의자 이동을 별도 음원군으로 관리한다.
- 바닥 재질과 이동 속도에 따라 발걸음 음량·간격·고주파 성분을 조절한다.
- NPC 소리는 거리 감쇠와 좌우 위치를 적용하고 교사 UI 소리는 비공간 음향으로 유지한다.
- 같은 효과음의 반복감을 줄이기 위해 피치·음량·샘플 시작점을 좁은 범위에서 변형한다.

## 6. 검증과 성능

- 외형 중복도, 동시 제스처 중복도, 교사 시선 정렬도, UI 가독성을 자동 측정한다.
- 얼굴 근접 캡처, 교실 전체 캡처, 교사 이동 중 캡처를 회귀 증거로 남긴다.
- 15명 동시 Animator·IK·AU 업데이트의 CPU와 스킨드 메시 비용을 프로파일링한다.
- 화면 밖 학생은 Animator와 얼굴 업데이트 빈도를 낮추되 시나리오 상태는 유지한다.

## 7. 촘촘한 구현 백로그

### P0: 훈련 루프와 상태 일관성

- `TRN-01` 관찰 → 대응 선택 → 학생 반응 → 디브리핑의 진입·종료·중단 조건을 상태표로 고정한다.
- `TRN-02` 버튼 선택, 자유 발화, 모드 전환이 동시에 들어올 때 입력 우선순위와 중복 제출 방지를 구현한다.
- `TRN-03` 학생의 감정, 각성, 지배감, 신뢰도, 과제 참여도를 매 턴 전후 스냅샷으로 저장한다.
- `TRN-04` 좋은 대응의 즉시 효과와 누적 효과를 분리하고, 한 번의 선택으로 위기가 종료되지 않게 한다.
- `TRN-05` 실패 후 회복적 재시도 경로와 안전 위기 발생 시 강제 종료 경로를 별도로 둔다.
- 완료 기준: 같은 시나리오를 선택형·자유대화형으로 각각 10회 반복해 상태 누락, 이중 입력, 진행 정지가 0건이다.

### P0: LLM 상호작용과 안전

- `LLM-01` OpenRouter 요청 모델, 타임아웃, 최대 토큰, 온도 설정을 환경 설정으로 분리한다.
- `LLM-02` 시스템 프롬프트에 학생 프로필, 현재 정서 벡터, 직전 교사 대응, 금지 응답 규칙을 구조화해 전달한다.
- `LLM-03` 응답을 학생 발화, 정서 변화량, 제스처 의도, 근거 태그로 나눈 구조화 JSON으로 검증한다.
- `LLM-04` JSON 오류, 타임아웃, 429, 네트워크 단절 시 로컬 대사 라이브러리로 복구한다.
- `LLM-05` 교사 입력과 모델 출력에서 개인정보, 자해·타해 위험, 낙인 표현을 탐지하고 안전 경로로 전환한다.
- `LLM-06` API 키는 프로젝트·로그·빌드에 포함하지 않고 OS 환경 변수 또는 로컬 비밀 설정에서만 읽는다.
- 완료 기준: 정상, 느린 응답, 잘못된 JSON, 429, 오프라인의 5개 시나리오에서 세션이 중단되지 않고 비밀값이 로그에 0회 노출된다.

### P0: 정서·AU·행동 전환

- `AFT-01` valence·arousal·dominance 범위와 턴당 최대 변화량을 정의한다.
- `AFT-02` AU01/02, AU04, AU05/07, AU12, AU15, AU20/23, AU25/26의 최소 조합을 표정 프리셋으로 만든다.
- `AFT-03` 대사 시작·강조·종료 지점에 AU와 고개·손 제스처의 시간축을 동기화한다.
- `AFT-04` 상충 AU, 과도한 좌우 비대칭, 눈꺼풀 관통, 입술 떨림을 제한한다.
- `AFT-05` 문제행동 강도를 0~1 연속값으로 두고 빈도·범위·속도·회복 지연에 각각 연결한다.
- `AFT-06` 같은 정서 상태에서도 최소 3개의 제스처 후보와 반복 방지 이력을 사용한다.
- 시작값: AU 전환 0.25~0.8초, 정서 최대 턴 변화량 0.42. 테스트에서 표정 튐이 보이면 전환 시간을 0.1초씩 늘리고, 교사 대응 차이가 약하면 최대 변화량을 0.05씩 늘린다.
- 완료 기준: 30초 관찰에서 동일 동작 완전 동기화가 없고, 평가자 5명 중 4명 이상이 감정 방향을 텍스트 없이 구분한다.

### P0: 시선·대면 대화·말풍선

- `GAZ-01` 눈동자, 머리, 흉곽 회전을 분리하고 각 관절의 제한각과 반응 지연을 둔다.
- `GAZ-02` 교사 이동 속도·거리·발화 여부에 따라 시선 추적 확률을 학생별로 다르게 한다.
- `GAZ-03` 정면 대화 모드에서 카메라를 학생 눈높이에 맞추고 상체를 세우되 순간이동처럼 보이지 않게 전환한다.
- `GAZ-04` 말풍선 앵커를 머리 위 월드 좌표에 두고 얼굴 가림, 화면 밖 이탈, HUD 중첩을 회피한다.
- `GAZ-05` 말풍선 텍스트는 SDF 폰트, 반투명 배경, 최대 폭, 줄바꿈, 긴 문장 스크롤 규칙을 갖는다.
- 완료 기준: 교사 좌우 이동 후 선택 학생 6명의 평균 시선 정렬도가 0.82 이상이고, 16:9·16:10·울트라와이드에서 얼굴 가림이 0건이다.

### P1: 한국 초등학교 환경과 학생 다양성

- `ENV-01` 전자칠판, 전면 수납장, 후면 게시판, 창호, 바닥, 천장 조명, 벽 선풍기를 개별 프리팹으로 분리한다.
- `ENV-02` 책상 폭·높이·상판 두께, 열 간격, 통로 폭을 참고 사진 비율표로 관리한다.
- `ENV-03` 게시물은 각각 별도 에셋으로 배치하고 회전·높이·간격에 작은 편차를 주어 과도한 정렬을 피한다.
- `ENV-04` 가방, 통신문, 태블릿, 필통, 물병을 좌석별 확률로 배치하고 가방은 책상 고리에 걸리게 한다.
- `ENV-05` 바닥·벽·목재·금속·플라스틱에 PBR 재질과 사용 흔적의 강도 프리셋을 둔다.
- `NPC-01` 얼굴형 7종 이상, 의상 색 8종 이상, 자연 흑발·진갈색 헤어 프로필 5종을 좌석 제약으로 분산한다.
- `NPC-02` Rocketbox 기본 아동 메시 한계를 보완할 추가 한국 아동 베이스 메시 도입 여부를 별도 라이선스 검토한다.
- 완료 기준: 전체 교실 캡처에서 동일 얼굴·의상·동작이 인접 좌석 3개 이상 연속되지 않고, 게시물이 하나의 통 이미지처럼 보이지 않는다.

### P1: 피드백, 오디오, 접근성

- `UX-01` 모든 클릭은 시각적 눌림과 짧은 클릭음을 함께 제공하고 비활성 이유를 텍스트로 표시한다.
- `UX-02` 관찰 정보는 학생 신호, 교사 발화, 시스템 피드백을 색·위치뿐 아니라 라벨로도 구분한다.
- `UX-03` 자막 크기, UI 배율, 고대비, 모션 감소, 음량 분리, 키 재지정을 설정에 넣는다.
- `AUD-01` 버튼 클릭 3종, 실내화 좌우 발걸음 4종 이상, 의자·책상 접촉음을 반복 방지 큐로 재생한다.
- `AUD-02` 학생 발화는 공간 음향, 교사 UI와 안내는 비공간 음향으로 믹싱 우선순위를 분리한다.
- 완료 기준: 신규 사용자가 설명 없이 2분 안에 첫 대응을 완료하고, 관찰자 10명 중 8명 이상이 선택 결과의 원인을 설명한다.

### P1: 평가·디브리핑·연구 데이터

- `EVA-01` 감정 인정, 선택권, 요구 조절, 원인 탐색, 안전 확보를 독립 점수와 근거 문장으로 저장한다.
- `EVA-02` 점수만 제시하지 않고 교사 입력 → 학생 신호 → 상태 변화의 인과 타임라인을 보여준다.
- `EVA-03` 정답 하나를 강제하지 않고 상황 적합성, 순서, 누적 맥락을 반영한다.
- `EVA-04` 세션 ID, 이벤트 시각, 상태 전후값, 모델 버전, 프롬프트 버전을 익명 로그로 남긴다.
- `EVA-05` 연구용 내보내기에서 자유 발화 원문 포함 여부를 동의·비식별 정책으로 제어한다.
- 완료 기준: 모든 점수 항목이 재현 가능한 이벤트 근거를 가지며, 동일 로그 재생 시 최종 점수와 상태가 일치한다.

## 8. 플레이 경험 5요소 점검

| 요소 | 현재 목표 | 검증 질문 |
|---|---|---|
| 명료성 | 학생 신호와 선택 결과의 원인이 보인다 | 신규 사용자가 2분 내 다음 행동을 예측하는가 |
| 동기 | 대응이 학생의 누적 상태와 디브리핑에 남는다 | 실패 뒤 다른 전략으로 재시도하는가 |
| 반응성 | 버튼·자유 발화·이동 입력이 즉시 확인된다 | 입력이 무시되거나 이중 처리되지 않는가 |
| 만족감 | 시각·소리·학생 반응의 최소 2채널 피드백이 있다 | 점수를 읽지 않아도 대응 효과를 느끼는가 |
| 적합성 | 한국 초등교실, 절제된 UI, 자연스러운 학생 행동을 유지한다 | 화면·음향·대사가 연구용 훈련 맥락에 맞는가 |

## 9. 권장 실행 순서

1. P0 상태 머신과 LLM 안전·복구를 먼저 고정한다.
2. AU·제스처·시선의 시간축을 하나의 행동 오케스트레이터로 연결한다.
3. 정면 대화와 말풍선 충돌 회피를 해상도별로 검증한다.
4. 환경·학생 에셋 다양성을 확장하고 성능 예산을 측정한다.
5. 디브리핑 근거와 연구 로그 재현성을 완성한다.
6. 신규 사용자, 스트레스 입력, 숙련자, 악용, 관찰자 가독성의 5개 플레이테스트를 수행한다.
