# Contributing

This is a research prototype. Keep changes small, reproducible, and evidence-backed.

1. Create a focused branch from the default branch.
2. Preserve Unity `.meta` files.
3. Make generated-scene changes in the builder as well as in any serialized output.
4. Add or update a test when changing a behavioral contract or fixing a subtle regression.
5. Run the Unity EditMode suite before requesting review.
6. Include screenshots for visual changes and identify the scene, resolution, and build used.
7. Never commit API keys, raw student data, or unlicensed media.

Report safety-sensitive scenario changes with the instructional rationale and the expected trainee behavior. Avoid diagnostic claims about student characters.
